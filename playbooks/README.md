# Design OS

This folder turns the repo from an inspiration archive into a usable design operating system.

Use it when you are not trying to recreate a single reference, but to synthesize something that feels authored, culturally current, and distinct from default SaaS design.

## Start here

- `scene-kit.md` — the top-level philosophy and assembly logic
- `interaction-archetypes.md` — reusable implementation-grade mechanics
- `validation-rubric.md` — the manual QA sequence for interaction-heavy references
- `world-systems.md` — six visual worlds to build from
- `motion-grammar.md` — how motion should behave, not just how it should look
- `type-systems.md` — type roles, foundries, and pairings
- `asset-forge.md` — how to use generated visuals without shipping AI slop
- `component-philosophy.md` — what to standardize and what to keep custom
- `../data/design-os.json` — machine-readable version for coding agents

## Core rule

Do not build one universal brand shell.
Build a stable behavioral foundation and multiple authored world systems on top of it.
