# Asset Forge

Use AI generation and procedural tools as a materials pipeline, not as a substitute for composition.

## What to generate

Good raw material:

- paper texture maps
- scan noise
- tape and sticker sheets
- scribble sets
- halftone overlays
- thermal receipt textures
- weird mascots
- warped fashion stills
- icon debris
- object materials and metallic gradients
- diagram fragments and annotation marks

Bad raw material:

- full hero layouts
- complete landing page compositions
- final icon systems
- polished product UI screens
- long-form illustrations that no human recomposes

## Workflow

1. Generate many raw assets
2. Curate aggressively
3. Crop, mask, vectorize, redraw, or simplify
4. Combine generated material with authored SVG and type
5. Animate selectively
6. Keep one or two signature materials per page, not ten

## Material packs to maintain

- `Scribbles` — arrows, circles, strikethroughs, underlines, notes
- `Stickers` — stamps, badges, labels, tabs, warning chips
- `Scans` — paper, xerox, dust, print bleed, fold marks
- `Chromes` — metallic gradients, glass distortions, iridescent panels
- `Artifacts` — tickets, access passes, merch tags, archive cards
- `Diagrams` — system arrows, maps, callouts, node lines, legends

## Procedural helpers worth using

- Rough.js for hand-drawn marks and imperfect rules
- p5.js for generated backgrounds and reactive patterns
- SVG filters for blur, displacement, grain, turbulence, glow
- Matter.js for piles, stacks, and drag physics
- Rive for interactive illustrated assets with clean state transitions

## The smell test

If the asset feels like a generic AI image, it should be treated as source material only.
If it feels like a designed artifact after editing, it can go live.
